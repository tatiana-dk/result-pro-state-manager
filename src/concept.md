# Единое хранилище с реактивностью и атомарными единицами данных

state-manager будет представлять собой единое хранилище — один центральный объект, в котором собраны все атомарные единицы данных приложения. Каждая атомарная единица — это независимый реактивный атом, хранящий одно простое значение и умеющий уведомлять подписчиков о своём изменении.

## Причины выбора концепции

Единое хранилище выбрано, т.к. оно обеспечивает лёгкость отслеживание данных, простоту отладки. Реактивность позволяет автоматически обновлять UI при изменении данных. Атомарные единицы при изменении затрагивают только те части приложения, которые от них зависят.

## Сильные стороны:

- простота доступа: всё состояние приложения доступно в одном месте (store.name, store.count и т.д.), легко импортировать и использовать в любом файле без пропсов или контекста;
- точная реактивность: при изменении одного атома обновляются только те части приложения, которые реально зависят от него — нет лишних перерисовок всего UI, высокая производительность даже в больших приложениях;
- предсказуемость: каждое значение меняется только через .set(), нет случайных мутаций объекта, легче отлаживать и находить баги;
- гибкость комбинаций: легко создавать производные значения (селекторы), которые автоматически пересчитываются из нескольких атомов, при этом они остаются лёгкими и без лишнего кода;
- независимость частей: разные модули приложения могут работать со своими атомами, не боясь сломать чужое состояние, но при этом всё собрано в едином хранилище для удобства;
- лёгкость тестирования: каждый атом можно тестировать отдельно, подменять значения или подписки без моков всего хранилища.
