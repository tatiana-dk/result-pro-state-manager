# Контракты ядра:

## Создание места для данных:

— Создаём атом — ячейку с именем и начальным значением.
— Атомы хранятся в едином хранилище под своими ключами.

## Чтение состояния:

— У каждого атома есть метод get(), который возвращает текущее значение.

## Обновление состояния:

— У каждого атома есть метод set(newValue), который меняет значение и автоматически уведомляет подписчиков.

## Уведомление об изменениях:

— При изменении атома вызываются все функции, подписанные на этот атом.
— Другие атомы не уведомляются (изоляция).

## Подписка на изменения:

— У атома есть метод subscribe(listener).
— listener — это функция, которая получает новое значение при изменении.

## Отписка:

— subscribe возвращает функцию unsubscribe(). Вызови её, чтобы перестать получать обновления.

## Ограничения/правила:

— Атомы идентифицируются по уникальным ключам.
— Изменение атома происходит только через set().
— Подписчики вызываются синхронно сразу после set().

## Примеры использования:

- Создал хранилище -> Создал единицу данных -> Получил значение -> Возвращает заданное значение;
- Создал хранилище -> Создал единицу данных -> Установил значение -> Получил значение -> Возвращает новое значение;
- Создал хранилище -> Создал единицу данных -> Подписался -> Установил значение -> Подписчик получил уведомление -> Отписался -> Обновил значение -> Подписчик больше не вызывается;
